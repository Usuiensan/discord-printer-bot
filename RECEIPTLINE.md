# ReceiptLine 対応リファレンス

この bot で実装している ReceiptLine 記法の対応範囲です。OFSC ReceiptLine 仕様のうち、印刷でよく使う範囲を実装しています。

## 使い方

通常印刷:

````text
!receiptline
```
{width:*,2,10}
ビール | 2| ¥1,300
千鳥コース | 2| ¥17,280
--------------------------------
{width:*,10}
^合計 | ^¥18,580
{code:https://example.com; option:qrcode,6,m}
```
````

短縮形:

```text
!rl
{width:*,10}
商品 | 1,000
```

プレビュー:

````text
!preview
!receiptline
```
{width:*,10}
商品 | 1,000
```
````

## 文書とコードブロック

- `!receiptline` または `!rl` に続く本文を ReceiptLine 文書として解釈します。
- Markdown コードブロックで囲めます。
- コードブロックの言語名は `receiptline` または `rl` でも省略でも構いません。
- 空行は空行として出力します。

## 列

列区切り:

```text
左 | 中央 | 右
```

列の位置揃えは、縦線 `|` の前後の空白で決まります。

| 書き方 | 揃え |
| --- | --- |
| `文字|` / `文字 |` | 左揃え |
| ` 文字|` | 右揃え |
| ` 文字 |` | 中央揃え |

列幅を超える文字列は、既定では折り返します。`{text:nowrap}` の場合は列幅で切り詰めます。

## プロパティ

プロパティは `{key:value; key:value}` の形式で、1行に1つ書きます。短縮キーも使えます。

### width / w

列幅を指定します。

```text
{width:*,2,10}
商品名 | 2| 1,000
```

対応値:

- `*`: 残り幅を自動配分
- `auto`: すべての列を自動配分
- `0`: その列を出力しない
- `1` 以上の整数: 固定桁数

既定値は `*` です。印字幅は 32 桁相当として計算します。

### border / b

列間を指定します。

```text
{border:space}
{border:none}
{border:line}
{border:2}
```

対応値:

- `space`: 1桁空白。既定値
- `none` / `0`: 列間なし
- `line`: 縦線 `|` を列間に出力
- `2`: 2桁空白

### align / a

行全体の揃えを指定します。行幅が 32 桁未満のときに作用します。

```text
{align:left}
{align:center}
{align:right}
```

対応値:

- `left`
- `center`
- `right`

不正値は `center` として扱います。

### text / t

列内の折り返しを指定します。

```text
{text:wrap}
{text:nowrap}
```

対応値:

- `wrap`: 折り返す。既定値
- `nowrap`: 折り返さず、列幅で切り詰める

### image / i

PNG base64 画像を印刷します。

```text
{image:iVBORw0KGgo...}
```

- 空白や改行を含む base64 は結合して扱います。
- 画像は `PRINT_WIDTH_DOTS` 内に収まるように既存画像印刷処理で変換します。
- プレビューでは `[ReceiptLine画像]` と表示します。

### code / c

バーコードまたは二次元コードを印刷します。

```text
{code:https://example.com; option:qrcode,6,m}
{code:490123456789; option:jan,hri}
{code:ABC123; option:code39,3,72,hri}
```

`option` が同じ行にある場合、その設定を反映してから `code` を出力します。

### option / o

`code` の種類とパラメータを指定します。

対応コード種別:

- `qrcode` / `qr`
- `upc`
- `ean` / `jan`
- `code39`
- `itf`
- `codabar` / `nw7`
- `code93`
- `code128`

対応オプション:

- `hri`: 1次元バーコードの人間可読文字を下に出す
- `nohri`: 人間可読文字を出さない。既定値
- `l` / `m` / `q` / `h`: QR 誤り訂正レベル
- 数値 `2` から `8`: バーコード幅、または QR セルサイズ
- 数値 `24` から `240`: バーコード高さ

`ean` / `jan` はデータ桁数から `jan8` または `jan13` を選びます。`upc` は `upc_a` として扱います。

### comment / _

コメントです。出力しません。

```text
{comment:ここは印刷されない}
{_:短縮コメント}
```

### command / x

安全のため無視します。デバイス固有コマンドが必要な場合は、許可ユーザーのみ使える raw ESC/POS 機能を使ってください。

## 水平線とカット

水平線:

```text
--------------------------------
```

- `-` だけの行は、32 桁の水平線として出力します。

カット:

```text
================================
```

- `=` だけの行は、その場で `CUT_MODE` に従ってカットします。
- 通常の Discord 1投稿の最後にも従来どおりカットが入ります。

## 文字装飾

対応している装飾:

| 記法 | 効果 |
| --- | --- |
| `_文字_` | 下線 |
| `"文字"` | 強調 |
| `` `文字` `` | 白黒反転 |
| `^文字^` | 横倍角 |

装飾は列内でトグルします。プレビューでは装飾記号を除去したテキストとして表示します。

## エスケープ

対応しているエスケープ:

| 記法 | 出力 |
| --- | --- |
| `\n` | 改行文字 |
| `\xNN` | 16進数コードの文字 |
| `\|` | 縦線 |
| `\{` / `\}` | 中括弧 |
| `\\` | バックスラッシュ |
| `¥` 始まり | `\` と同じ扱い |
| `~` | 空白 |

## プレビュー表示

- `!preview` と組み合わせると PNG プレビューを返します。
- 画像は `[ReceiptLine画像]` と表示します。
- コードは `[QRCODE: データ]` や `[CODE128: データ]` のように表示します。
- カットは `[CUT]` と表示します。

## 未対応・制限

- ReceiptLine 仕様の完全実装ではありません。
- `border:line` の外枠罫線生成は簡易対応です。列間は `|`、水平線は `-` で表現します。
- `{command:...}` は無視します。
- 画像は PNG base64 前提です。
- 表示幅は半角1桁、非ASCIIを概ね全角2桁として計算します。
